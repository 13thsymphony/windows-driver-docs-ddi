# DXGKMDT_OPM_ACP_AND_CGMSA_SIGNALING structure



## -description
<p>The DXGKMDT_OPM_ACP_AND_CGMSA_SIGNALING structure describes how the signal that goes through the physical connector that is associated with the protected output object is protected.</p>


## -syntax

````
typedef struct _DXGKMDT_OPM_ACP_AND_CGMSA_SIGNALING {
  DXGKMDT_OPM_RANDOM_NUMBER rnRandomNumber;
  ULONG                     ulStatusFlags;
  ULONG                     ulAvailableTVProtectionStandards;
  ULONG                     ulActiveTVProtectionStandard;
  ULONG                     ulReserved;
  ULONG                     ulAspectRatioValidMask1;
  ULONG                     ulAspectRatioData1;
  ULONG                     ulAspectRatioValidMask2;
  ULONG                     ulAspectRatioData2;
  ULONG                     ulAspectRatioValidMask3;
  ULONG                     ulAspectRatioData3;
  ULONG                     ulReserved2[4];
  ULONG                     ulReserved3[4];
} DXGKMDT_OPM_ACP_AND_CGMSA_SIGNALING;
````


## -struct-fields
<dl>

### -field rnRandomNumber

<dd>
<p>A <a href="..\d3dkmdt\ns-d3dkmdt--dxgkmdt-opm-random-number.md">DXGKMDT_OPM_RANDOM_NUMBER</a> structure that contains a protected output object's 128-bit cryptographically secure random number. This random number is generated by an application and supplied to the display miniport driver in a call to the driver's <a href="..\dispmprt\nc-dispmprt-dxgkddi-opm-get-copp-compatible-information.md">DxgkDdiOPMGetCOPPCompatibleInformation</a> function. This random number is supplied to the driver in the <b>rnRandomNumber</b> member of the <a href="..\d3dkmdt\ns-d3dkmdt--dxgkmdt-opm-copp-compatible-get-info-parameters.md">DXGKMDT_OPM_COPP_COMPATIBLE_GET_INFO_PARAMETERS</a> structure.</p>
</dd>

### -field ulStatusFlags

<dd>
<p>A bitwise OR combination of the values from the <a href="..\d3dkmdt\ne-d3dkmdt--dxgkmdt-opm-status.md">DXGKMDT_OPM_STATUS</a> enumeration that indicates the status of a protected output.</p>
</dd>

### -field ulAvailableTVProtectionStandards

<dd>
<p>A valid bitwise OR combination of values from the <a href="..\d3dkmdt\ne-d3dkmdt--dxgkmdt-opm-protection-standard.md">DXGKMDT_OPM_PROTECTION_STANDARD</a> enumeration that indicates the types of television signals that the physical connector that is associated with a protected output can carry.</p>
</dd>

### -field ulActiveTVProtectionStandard

<dd>
<p>A valid bitwise OR combination of values from the <a href="..\d3dkmdt\ne-d3dkmdt--dxgkmdt-opm-protection-standard.md">DXGKMDT_OPM_PROTECTION_STANDARD</a> enumeration that indicates the types of television signals that the physical connector that is associated with a protected output currently carries.</p>
</dd>

### -field ulReserved

<dd>
<p>Reserved. Must be set to zero. </p>
</dd>

### -field ulAspectRatioValidMask1

<dd>
<p>A mask value that indicates the valid bits in the following <b>ulAspectRatioData1</b> member.</p>
</dd>

### -field ulAspectRatioData1

<dd>
<p>32-bit data that indicates the aspect ratio value that is currently set for the active protection standard.</p>
</dd>

### -field ulAspectRatioValidMask2

<dd>
<p>A mask value that indicates the valid bitfields in the following <b>ulAspectRatioData2</b> member.</p>
</dd>

### -field ulAspectRatioData2

<dd>
<p>32-bit data for additional aspect ratio information that is used for specific protection standards. This data can be used to read End and Q0 values for EIA-608-B, or active format description for CEA-805-A Type B packets.</p>
</dd>

### -field ulAspectRatioValidMask3

<dd>
<p>A mask value that indicates the valid bitfields in the following <b>ulAspectRatioData3</b> member.</p>
</dd>

### -field ulAspectRatioData3

<dd>
<p>32-bit data for additional aspect ratio information that is used for specific protection standards. This data can be used to read End and Q0 values for EIA-608-B, or active format description for CEA-805-A Type B packets.</p>
</dd>

### -field ulReserved2

<dd>
<p>Reserved. Must be set to zero. </p>
</dd>

### -field ulReserved3

<dd>
<p>Reserved. Must be set to zero. </p>
</dd>
</dl>
