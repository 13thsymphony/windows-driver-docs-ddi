# PFND3D10DDI_SETCONSTANTBUFFERS callback



## -description
<p>The <b>CsSetConstantBuffers</b> function sets constant buffers for a compute shader. </p>


## -prototype

````
PFND3D10DDI_SETCONSTANTBUFFERS CsSetConstantBuffers;

VOID APIENTRY CsSetConstantBuffers(
  _In_       D3D10DDI_HDEVICE   hDevice,
  _In_       UINT               StartBuffer,
  _In_       UINT               NumBuffers,
  _In_ const D3D10DDI_HRESOURCE *phBuffers
)
{ ... }
````


## -parameters
<dl>

### -param hDevice [in]

<dd>
<p> A handle to the display device (graphics context).</p>
</dd>

### -param StartBuffer [in]

<dd>
<p> The starting constant buffer to set. </p>
</dd>

### -param NumBuffers [in]

<dd>
<p> The total number of buffers to set. </p>
</dd>

### -param phBuffers [in]

<dd>
<p> An array of handles to the constant buffers, beginning with the buffer that <b>StartBuffer</b> specifies.</p>
</dd>
</dl>

## -returns
<p>None</p>

<p>The driver can use the <a href="..\d3d10umddi\nc-d3d10umddi-pfnd3d10ddi-seterror-cb.md">pfnSetErrorCb</a> callback function to set an error code. For more information about setting error codes, see the following Remarks section.</p>

## -remarks
<p>Buffers that the <b>CsSetConstantBuffers</b> function specifies are created with the D3D10_BIND_CONSTANT_BUFFER flag. </p>

<p>The driver should not encounter any error, except for D3DDDIERR_DEVICEREMOVED. Therefore, if the driver passes any error, except for D3DDDIERR_DEVICEREMOVED, in a call to the <a href="..\d3d10umddi\nc-d3d10umddi-pfnd3d10ddi-seterror-cb.md">pfnSetErrorCb</a> function, the Direct3D runtime determines that the error is critical. Even if the device is removed, the driver is not required to return D3DDDIERR_DEVICEREMOVED; however, if device removal interferes with the operation of <b>CsSetConstantBuffers</b> (which typically should not happen), the driver can return D3DDDIERR_DEVICEREMOVED.</p>
