# IPrintOemPS::Command method



## -description
<p>The <code>IPrintOemPS::Command</code> method is used by rendering plug-ins for the Microsoft PostScript printer driver, in order to insert PostScript commands into the print job's data stream.</p>


## -syntax

````
HRESULT Command(
        PDEVOBJ pdevobj,
        DWORD   dwIndex,
        PVOID   pData,
        DWORD   cbSize,
  [out] DWORD   *pdwResult
);
````


## -parameters
<dl>

### -param <i>pdevobj</i> 

<dd>
<p>Caller-supplied pointer to a <a href="https://msdn.microsoft.com/library/windows/hardware/ff547573">DEVOBJ</a> structure.</p>
</dd>

### -param <i>dwIndex</i> 

<dd>
<p>Caller-supplied value representing the current injection point within the print job's data stream. For more information, see the following Remarks section.</p>
</dd>

### -param <i>pData</i> 

<dd>
<p>Not used.</p>
</dd>

### -param <i>cbSize</i> 

<dd>
<p>Not used.</p>
</dd>

### -param <i>pdwResult</i> [out]

<dd>
<p>Receives a method-supplied value indicating the result of the insertion operation. If the insertion succeeds, this value should be ERROR_SUCCESS. If the insertion fails, the value should be one of the error codes defined in winerror.h. If the method does not attempt to insert data for a specified <i>dwIndex</i> value, the value should be ERROR_NOT_SUPPORTED.</p>
</dd>
</dl>

## -returns
<p>The method must return one of the following values.</p><dl>
<dt><b>S_OK</b></dt>
</dl><p>The operation succeeded.</p><dl>
<dt><b>E_FAIL</b></dt>
</dl><p>The operation failed</p><dl>
<dt><b>E_NOTIMPL</b></dt>
</dl><p>The method is not implemented.</p>

<p> </p>

## -remarks
<p>The <code>IPrintOemPS::Command</code> method is used to insert PostScript commands into the data stream that is generated by the <a href="NULL">Microsoft PostScript Printer Driver</a>. If you want to provide this method, you must define it in a rendering plug-in.</p>

<p>When the <code>IPrintOemPS::Command</code> method is called, its <i>dwIndex</i> parameter specifies the current injection point within the print job's data stream. The method should use this value to determine what operation, if any, to perform. Index values, which are defined in wingdi.h, are listed in the following table.</p>

<p>PSINJECT_BEGINDEFAULTS</p>

<p>After the printer driver sends  <b>%%BeginDefaults</b> to the data stream.</p>

<p>PSINJECT_BEGINPAGESETUP</p>

<p>After the printer driver sends <b>%%BeginPageSetup</b> to the data stream.</p>

<p>PSINJECT_BEGINPROLOG</p>

<p>After the printer driver sends <b>%%BeginProlog</b> to the data stream.</p>

<p>PSINJECT_BEGINSETUP</p>

<p>After the printer driver sends <b>%%BeginSetup</b> to the data stream.</p>

<p>PSINJECT_BEGINSTREAM</p>

<p>Before the printer driver sends the first byte of the data stream.</p>

<p>PSINJECT_BOUNDINGBOX</p>

<p>When the printer driver is ready to insert <b>%%BoundingBox:</b> into the data stream. Data supplied by <code>IPrintOemPS::Command</code> replaces the driver's <b>%%BoundingBox:</b> comment.</p>

<p>PSINJECT_COMMENTS</p>

<p>Before the printer driver sends <b>%%EndComments</b> to the data stream.</p>

<p>PSINJECT_DLFONT</p>

<p>Before the print driver downloads a soft font.</p>

<p>PSINJECT_DOCNEEDEDRES</p>

<p>After the printer driver sends <b>%%DocumentNeededResources:</b> to the data stream, along with resource information. If the <code>IPrintOemPS::Command</code> method sends resource information, it is appended to the stream after the driver's resource information. Entry format is as follows:</p>

<p>%%+ 
        <i>ResourceTypeResourceNames</i></p>

<p>The printer driver sends <b>%%DocumentNeededResources: </b>after sending <b>%%Trailer</b> section.</p>

<p>PSINJECT_DOCSUPPLIEDRES</p>

<p>After the printer driver sends <b>%%DocumentSuppliedResources:</b> to the data stream, along with resource information. If the <code>IPrintOemPS::Command</code> method sends resource information, it is appended to the stream after the driver's resource information. Entry format is as follows:</p>

<p>
        %%+ 
        <i>ResourceTypeResourceNames</i></p>

<p>The printer driver sends <b>%%DocumentSuppliedResources:</b> after sending <b>%%Trailer</b> section.</p>

<p>PSINJECT_DOCUMENTPROCESSCOLORS</p>

<p>When the printer driver is ready to insert <b>%%DocumentProcessColors:</b><i>color</i> into the data stream. Data supplied by <code>IPrintOemPS::Command</code> replaces the driver's <b>%%DocumentProcessColors:</b><i>color</i> comment.</p>

<p>PSINJECT_DOCUMENTPROCESSCOLORSATEND</p>

<p>When the printer driver is ready to insert <b>%%DocumentProcessColors:</b>(atend) into the data stream. Data supplied by <code>IPrintOemPS::Command</code> replaces the driver's <b>%%DocumentProcessColors:</b>(atend) comment.</p>

<p>PSINJECT_ENDDEFAULTS</p>

<p>Before the printer driver sends <b>%%EndDefaults</b> to the data stream.</p>

<p>PSINJECT_ENDPAGECOMMENTS</p>

<p>Before the printer driver sends <b>%%EndPageComments</b> to the data stream.</p>

<p>PSINJECT_ENDPAGESETUP</p>

<p>Before the printer driver sends <b>%%EndPageSetup</b> to the data stream.</p>

<p>PSINJECT_ENDPROLOG</p>

<p>Before the printer driver sends <b>%%EndProlog</b> to the data stream.</p>

<p>PSINJECT_ENDSETUP</p>

<p>Before the printer driver sends <b>%%EndSetup</b> to the data stream.</p>

<p>PSINJECT_ENDSTREAM</p>

<p>After the printer driver sends the last byte of the data stream.</p>

<p>PSINJECT_EOF</p>

<p>After the printer driver sends <b>%%EOF</b> to the data stream.</p>

<p>PSINJECT_ORIENTATION</p>

<p>When the printer driver is ready to insert <b>%%Orientation:</b> into the data stream. Data supplied by <code>IPrintOemPS::Command</code> replaces the driver's <b>%%Orientation:</b> comment.</p>

<p>PSINJECT_PAGEBBOX</p>

<p>When the printer driver is ready to insert <b>%%PageBoundingBox:</b> into the data stream. Data supplied by <code>IPrintOemPS::Command</code> replaces the driver's <b>%%PageBoundingBox:</b> comment.</p>

<p>PSINJECT_PAGENUMBER</p>

<p>When the printer driver is ready to insert <b>%%Page:</b> into the data stream. Data supplied by <code>IPrintOemPS::Command</code> replaces the driver's <b>%%Page:</b> comment.</p>

<p>PSINJECT_PAGEORDER</p>

<p>When the printer driver is ready to insert <b>%%PageOrder:</b> into the data stream. Data supplied by <code>IPrintOemPS::Command</code> replaces the driver's <b>%%PageOrder:</b> comment.</p>

<p>PSINJECT_PAGES</p>

<p>When the printer driver is ready to insert <b>%%Pages:</b><i>nnn</i> into the data stream. Data supplied by <code>IPrintOemPS::Command</code> replaces the driver's <b>%%Pages:</b><i>nnn</i> comment.</p>

<p>PSINJECT_PAGESATEND</p>

<p>When the printer driver is ready to insert <b>%%Pages:</b>(atend) into the data stream. Data supplied by <code>IPrintOemPS::Command</code> replaces the driver's <b>%%Pages:</b>(atend) comment.</p>

<p>PSINJECT_PAGETRAILER</p>

<p>After the printer driver sends <code>%%PageTrailer</code> to the data stream.</p>

<p>PSINJECT_PLATECOLOR</p>

<p>When the printer driver is ready to insert <b>%%PlateColor:</b><i>color</i> into the data stream. Data supplied by <code>IPrintOemPS::Command</code> replaces the driver's <b>%%PlateColor:</b><i>color</i> comment.</p>

<p>PSINJECT_PSADOBE</p>

<p>Before the printer driver sends <b>%!PS-Adobe</b> to the data stream.</p>

<p>PSINJECT_SHOWPAGE</p>

<p>Before the printer driver sends a <b>showpage</b> command.</p>

<p>PSINJECT_TRAILER</p>

<p>After the printer driver sends <b>%%Trailer</b> to the data stream.</p>

<p>PSINJECT_VMRESTORE</p>

<p>After the printer driver sends a <b>restore</b> command. (The <code>IPrintOemPS::Command</code> method must resend all resources it sent after the last PSINJECT_VMSAVE, if the resources are going to be reused.)</p>

<p>PSINJECT_VMSAVE</p>

<p>Before the printer driver sends a <b>save</b> command. (See PSINJECT_VMRESTORE.)</p>

<p> </p>

<p>The <code>IPrintOemPS::Command</code> method must call <a href="https://msdn.microsoft.com/library/windows/hardware/ff553103">IPrintOemDriverPS::DrvWriteSpoolBuf</a> to insert the PostScript commands.</p>

<p>If <code>IPrintOemPS::Command</code> methods are exported by multiple rendering plug-ins, the methods are called in the order that the plug-ins are specified for installation, every time that the printer driver reaches an injection point. The following rules apply:</p>

<p>If the injection point is one for which supplied data is appended to the data that the printer driver provides, each plug-in is called and each plug-in is allowed to append PostScript commands.</p>

<p>If the injection point is one for which supplied data replaces the data that the printer driver provides, the plug-ins are called until one <code>IPrintOemPS::Command</code> method supplies ERROR_SUCCESS for <i>dwResult</i>. No other plug-ins are called. If all plug-ins supply ERROR_NOT_SUPPORTED, the driver-supplied data is not replaced.</p>

<p>If the injection point is one for which supplied data replaces the data that the printer driver provides, and if an application has already replaced the driver-supplied data by using the <a href="https://msdn.microsoft.com/c0133355-fa3b-4117-bd38-b6a8b3898f94">POSTSCRIPT_INJECTION</a> printer escape function, no plug-ins are called.</p>
