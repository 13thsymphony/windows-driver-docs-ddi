# EVT_IDD_CX_MONITOR_QUERY_TARGET_MODES callback



## -description
<p><b>EVT_IDD_CX_MONITOR_QUERY_TARGET_MODES</b> is called by the OS to get a list of target modes supported by the driver for a monitor connected to the endpoint.</p>


## -prototype

````
EVT_IDD_CX_MONITOR_QUERY_TARGET_MODES EvtIddCxMonitorQueryTargetModes;

NTSTATUS EvtIddCxMonitorQueryTargetModes(
  _In_        IDDCX_MONITOR               MonitorObject,
  _In_  const IDARG_IN_QUERYTARGETMODES*  pInArgs,
  _Out_       IDARG_OUT_QUERYTARGETMODES* pOutArgs
)
{ ... }

typedef EVT_IDD_CX_MONITOR_QUERY_TARGET_MODES PFN_IDD_CX_MONITOR_QUERY_TARGET_MODES;
````


## -parameters
<dl>

### -param <i>MonitorObject</i> [in]

<dd>
<p>
                    
                A handle by the OS to identify the monitor to generate a list of target modes for.</p>
</dd>

### -param <i>pInArgs</i> [in]

<dd>
<p>
                    
                Input arguments used by <b>EVT_IDD_CX_MONITOR_QUERY_TARGET_MODES</b>.</p>
</dd>

### -param <i>pOutArgs</i> [out]

<dd>
<p>
                    
                Output arguments generated by <b>EVT_IDD_CX_MONITOR_QUERY_TARGET_MODES</b>.</p>
</dd>
</dl>

## -returns
<p>
(NTSTATUS) If the operation is successful, the callback function must return STATUS_SUCCESS, or another status value for which NT_SUCCESS(status) equals TRUE. Otherwise, an appropriate <a href="https://msdn.microsoft.com/7792201b-63bb-4db5-803d-2af02893d505">NTSTATUS</a> error code. 
                    </p>
