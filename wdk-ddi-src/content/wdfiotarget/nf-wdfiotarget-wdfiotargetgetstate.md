# WdfIoTargetGetState function



## -description
<p class="CCE_Message">[Applies to KMDF and UMDF]</p>
<p>The <b>WdfIoTargetGetState</b> method returns state information for a local or remote I/O target.</p>


## -syntax

````
WDF_IO_TARGET_STATE WdfIoTargetGetState(
  _In_ WDFIOTARGET IoTarget
);
````


## -parameters
<dl>

### -param <i>IoTarget</i> [in]

<dd>
<p>A handle to a local or remote I/O target object that was obtained from a previous call to <a href="..\wdfdevice\nf-wdfdevice-wdfdevicegetiotarget.md">WdfDeviceGetIoTarget</a> or <a href="..\wdfiotarget\nf-wdfiotarget-wdfiotargetcreate.md">WdfIoTargetCreate</a> or from a method that a specialized I/O target supplies.</p>
</dd>
</dl>

## -returns
<p><b>WdfIoTargetGetState</b> returns a <a href="..\wudfddi_types\ne-wudfddi-types--wdf-io-target-state.md">WDF_IO_TARGET_STATE</a>-typed value that indicates the state of the specified I/O target.</p>

<p>A bug check occurs if the driver supplies an invalid object handle.

</p>

## -remarks
<p>For more information about <b>WdfIoTargetGetState</b>, see <a href="wdf.controlling_a_general_i_o_target_s_state">Controlling a General I/O Target's State</a>. </p>

<p>For more information about I/O targets, see <a href="wdf.using_i_o_targets">Using I/O Targets</a>.</p>

<p>The following code example obtains state information for a USB I/O target.</p>
