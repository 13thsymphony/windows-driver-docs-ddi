# ZwLoadDriver function



## -description
<p>The <b>ZwLoadDriver</b> routine loads a driver into the system. </p>


## -syntax

````
NTSTATUS ZwLoadDriver(
  _In_ PUNICODE_STRING DriverServiceName
);
````


## -parameters
<dl>

### -param <i>DriverServiceName</i> [in]

<dd>
<p>Pointer to a counted Unicode string that specifies a path to the driver's registry key, <b>\Registry\Machine\System\CurrentControlSet\Services\</b><i>DriverName</i>, where <i>DriverName</i> is the name of the driver. </p>
</dd>
</dl>

## -returns
<p><b>ZwLoadDriver</b> returns STATUS_SUCCESS or an appropriate error NTSTATUS value. </p>

## -remarks
<p><b>ZwLoadDriver</b> dynamically loads a device or file system driver into the currently running system. </p>

<p>A minifilter should use <a href="https://msdn.microsoft.com/library/windows/hardware/ff543366">FltLoadFilter</a> instead of <b>ZwLoadDriver</b> to load a supporting minifilter.</p>

<p>For calls from kernel-mode drivers, the <b>Nt<i>Xxx</i></b> and <b>Zw<i>Xxx</i></b> versions of a Windows Native System Services routine can behave differently in the way that they handle and interpret input parameters. For more information about the relationship between the <b>Nt<i>Xxx</i></b> and <b>Zw<i>Xxx</i></b> versions of a routine, see <a href="https://msdn.microsoft.com/library/windows/hardware/ff565438">Using Nt and Zw Versions of the Native System Services Routines</a>.</p>
