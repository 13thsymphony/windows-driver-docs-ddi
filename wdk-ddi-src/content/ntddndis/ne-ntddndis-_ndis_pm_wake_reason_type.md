# _NDIS_PM_WAKE_REASON_TYPE Enumeration
The <b>NDIS_PM_WAKE_REASON_TYPE</b> enumeration identifies the type of wake-up event that was generated by the network adapter.

## Syntax
````
typedef enum _NDIS_PM_WAKE_REASON_TYPE { 
  NdisWakeReasonUnspecified               = 0x0000,
  NdisWakeReasonPacket                    = 0x0001,
  NdisWakeReasonMediaDisconnect           = 0x0002,
  NdisWakeReasonMediaConnect              = 0x0003,
  NdisWakeReasonWlanNLODiscovery          = 0x1000,
  NdisWakeReasonWlanAPAssociationLost     = 0x1001,
  NdisWakeReasonWlanGTKHandshakeError     = 0x1002,
  NdisWakeReasonWlan4WayHandshakeRequest  = 0x1003,
  NdisWakeReasonWwanRegisterState         = 0x2000,
  NdisWakeReasonWwanSMSReceive            = 0x2001,
  NdisWakeReasonWwanUSSDReceive           = 0x2002
} NDIS_PM_WAKE_REASON_TYPE, *PNDIS_PM_WAKE_REASON_TYPE;
````

## Constants

<table>
            
                <tr>
                    <td>NdisWakeReasonMediaConnect</td>
                    <td>The network adapter generated the wake-up event because it connected to the network media.</td>
                </tr>
            
                <tr>
                    <td>NdisWakeReasonMediaDisconnect</td>
                    <td>The network adapter generated the wake-up event because it disconnected from the network media.</td>
                </tr>
            
                <tr>
                    <td>NdisWakeReasonPacket</td>
                    <td>The network adapter generated the wake-up event because it received a packet that matched a wake-on-LAN (WOL) pattern.</td>
                </tr>
            
                <tr>
                    <td>NdisWakeReasonUnspecified</td>
                    <td>The type of wake-up event is not specified.</td>
                </tr>
            
                <tr>
                    <td>NdisWakeReasonWlan4WayHandshakeRequest</td>
                    <td>The 802.11 network adapter generated the wake-up event because it received the first frame of the IEEE 802.11i RSN 4-way handshake with the AP. This handshake is performed when the adapter authenticates with the AP.</td>
                </tr>
            
                <tr>
                    <td>NdisWakeReasonWlanAPAssociationLost</td>
                    <td>The 802.11 network adapter generated the wake-up event because it became disassociated with the access point (AP).</td>
                </tr>
            
                <tr>
                    <td>NdisWakeReasonWlanGTKHandshakeError</td>
                    <td>The 802.11 network adapter generated the wake-up event because it encountered an error during the IEEE 802.11i RSN group transient key (GTK) handshake with the AP.</td>
                </tr>
            
                <tr>
                    <td>NdisWakeReasonWlanNLODiscovery</td>
                    <td>The 802.11 network adapter generated the wake-up event because it detected a service set identifier (SSID) that was specified through a network list offload (NLO). 

For more information about NLO, see <a href="https://msdn.microsoft.com/528838AA-4002-4923-A71B-37ADEE9B8D07">Wi-Fi Network List Offload</a>.</td>
                </tr>
            
                <tr>
                    <td>NdisWakeReasonWwanRegisterState</td>
                    <td>The mobile broadband (MB) network adapter generated the wake-up event because its registration state to the MB Service has changed.</td>
                </tr>
            
                <tr>
                    <td>NdisWakeReasonWwanSMSReceive</td>
                    <td>The mobile broadband (MB) network adapter generated the wake-up event because the MB Service has to be notified about the receipt of a Short Message Service (SMS) message. The adapter generates this wake-up event either after the completion of a previously-issued <a href="https://msdn.microsoft.com/library/windows/hardware/ff569839">OID_WWAN_SMS_READ</a> query request, or the arrival of a new class-0 (flash/alert) message from the network provider as an event notification.</td>
                </tr>
            
                <tr>
                    <td>NdisWakeReasonWwanUSSDReceive</td>
                    <td>The mobile broadband (MB) network adapter generated the wake-up event because it received an Unstructured Supplementary Service Data (USSD) message.</td>
                </tr>
</table>
