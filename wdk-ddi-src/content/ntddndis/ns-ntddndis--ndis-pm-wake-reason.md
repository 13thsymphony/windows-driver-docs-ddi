# NDIS_PM_WAKE_REASON structure



## -description
<p>
<p>The <b>NDIS_PM_WAKE_REASON</b> structure specifies information about the wake-up event that was generated by a network adapter.</p>
</p>
<p>The <b>NDIS_PM_WAKE_REASON</b> structure specifies information about the wake-up event that was generated by a network adapter.</p>


## -syntax

````
typedef struct _NDIS_PM_WAKE_REASON {
  NDIS_OBJECT_HEADER       Header;
  ULONG                    Flags;
  NDIS_PM_WAKE_REASON_TYPE WakeReason;
  ULONG                    InfoBufferOffset;
  ULONG                    InfoBufferSize;
} NDIS_PM_WAKE_REASON, *PNDIS_PM_WAKE_REASON;
````


## -struct-fields
<dl>

### -field <b>Header</b>

<dd>
<p>The type, revision, and size of the <b>NDIS_PM_WAKE_REASON</b> structure. This member is formatted as an <a href="https://msdn.microsoft.com/library/windows/hardware/ff566588">NDIS_OBJECT_HEADER</a> structure.</p>
<p>The miniport driver must set the <b>Type</b> member of <b>Header</b> to NDIS_OBJECT_TYPE_DEFAULT. To specify the version of the <b>NDIS_PM_WAKE_REASON</b> structure, the driver must set the <b>Revision</b> member of <b>Header</b> to the following value: </p>
<p></p>
<dl>

### -field <a id="NDIS_PM_WAKE_REASON_REVISION_1"></a><a id="ndis_pm_wake_reason_revision_1"></a>NDIS_PM_WAKE_REASON_REVISION_1

<dd>
<p>Original version for NDIS 6.30 and later.</p>
<p>Set the <b>Size</b> member to NDIS_SIZEOF_PM_WAKE_REASON_REVISION_1.</p>
</dd>
</dl>
</dd>

### -field <b>Flags</b>

<dd>
<p>A <b>ULONG</b> value that contains a bitwise <b>OR</b> of flags. This member is reserved for NDIS.

</p>
</dd>

### -field <b>WakeReason</b>

<dd>
<p>An <a href="https://msdn.microsoft.com/library/windows/hardware/hh451607">NDIS_PM_WAKE_REASON_TYPE</a> enumeration value that specifies the reason why the network adapter generated the wake-up event.</p>
</dd>

### -field <b>InfoBufferOffset</b>

<dd>
<p>A <b>ULONG</b> value that specifies the offset, in units of bytes, from the beginning of the <b>NDIS_PM_WAKE_REASON</b> structure to the start of a buffer that contains additional information about the wake-up event.</p>
<div class="alert"><b>Note</b>  If the  <b>WakeReason</b> member is not set to NdisWakeReasonPacket, the miniport driver must set the <b>InfoBufferOffset</b> member to zero.</div>
<div> </div>
<div class="alert"><b>Note</b>  The offset to the buffer must be aligned on a 64-bit boundary.</div>
<div> </div>
</dd>

### -field <b>InfoBufferSize</b>

<dd>
<p>A <b>ULONG</b> value that specifies the length, in units of bytes, of the buffer that contains additional information about the wake-up event. </p>
<div class="alert"><b>Note</b>  If the  <b>WakeReason</b> member is not set to NdisWakeReasonPacket, the miniport driver must set the <b>InfoBufferSize</b> member to zero.</div>
<div> </div>
</dd>
</dl>

## -remarks
<p>The <b>NDIS_PM_WAKE_REASON</b> structure is used in the <a href="https://msdn.microsoft.com/library/windows/hardware/hh439808">NDIS_STATUS_PM_WAKE_REASON</a> status indication.</p>

<p>For more information about how to issue an <a href="https://msdn.microsoft.com/library/windows/hardware/hh439808">NDIS_STATUS_PM_WAKE_REASON</a> status indication, see <a href="NULL">Issuing NDIS Wake Reason Status Indications</a>.</p>
